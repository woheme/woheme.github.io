MarkDown总结

发表于 2015-06-05 | 分类于 Hexo

前言

markdown的语法不怎么难，但有很多细节还是需要注意的，另外就是好多markdown编辑器都对markdwon做了些扩展，导致一些功能的实现方式和使用方式都有差别。


MarkDown简介

Markdown 是一种轻量级标记语言，创始人为约翰·格鲁伯（John Gruber）。它允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML(或者HTML)文档”。[1]这种语言吸收了很多在电子邮件中已有的纯文本标记的特性。
——维基百科

MarkDwon语法

网上关于MarkDown的语法太多了，这里就不细说了

  • 在Hexo中需要注意摘要可以用<!-- more -->
  • 行内代码可以用两个反引号比如`println`
  • 在markdown插入图片可以使用yotuku，它可以直接和七牛相连接，通过拖拽粘贴图片瞬间就能得到图片链接，非常便捷
  • markdwon的语法可以参考
    1. markdown简明语法

关于markdown编辑器

写文章的目的是对自己做一个总结，总结不仅是发表在网上就算终点，而是希望能够把这些总结纳入自己已有的知识库比如EverNote中，但现在的markdown编辑器还没有把EverNote和个人博客发布结合的特别好的。目前主要用markdownpad,偶尔也会用用马克飞象。用markdownpad写好的文章发布到网上后我会用EverNote的浏览器剪裁工具剪裁到我的笔记本中。

markdown可以做有趣的事

  • 制作使用文档
    • readthedocs，使用最广的文档服务，如Scrapy 文档，支持Restructed和Markdown语言，其Markdown支持通过mkdocs来实现
  • 制作电子书(甚至可以像github那样开源)
    • gitbook，利用Markdown写电子书的工具，并提供免费托管。
  • 制作简历
  • 制作PPT
    • Landslide是基于Google的html5slides的一个Slide生成工具，可以将markdown, ReST 或者 textile文件转化成HTML5的slide。
